Under the hood

How available dates are calculated

Every time the picker loads, Slotora builds the list of bookable dates from your settings — in your site's timezone — so the calendar always reflects real capacity.

01

Start from lead time

Begin min_days ahead of today (default: 1 = next day).

02

Apply the cutoff

Past cutoff_hour (default 2 PM)? Push the start forward one more day.

03

Scan the window

Walk every day up to max_days ahead (default: 14).

04

Keep valid days

Include a day only if it's a working day and not a blackout date.

Then, per date

When a customer selects a date, an AJAX call returns that date's slots. Each slot's remaining capacity is checked live against the bookings table — a slot at its limit is shown as full and can't be chosen.

Installation & setup

WooCommerce must be installed and active first — Slotora depends on it and shows a notice if it's missing.

  1. Upload the plugin

    Copy the slotora-delivery-scheduler folder to /wp-content/plugins/, or install the ZIP from Plugins → Add New.

  2. Activate

    Go to Plugins → Installed Plugins and activate Slotora Delivery Scheduler. Activation creates the bookings table and sensible defaults.

  3. Configure your slots

    Open WooCommerce → Delivery Slots to set labels, working days, cutoff, lead time, time slots, and blackout dates.

  4. Test the checkout

    Add a product and visit checkout — the date picker and time-slot field appear under order information.

Extend it

For developers

Slotora ships with filters and actions so Pro — or your own code — can hook into availability and bookings without patching core files.

// Inject or filter the list of bookable dates (Y-m-d strings)
add_filter( 'slotora_available_dates', function( $dates ) {
    return $dates;
} );

// Return zone- or product-specific slots for a given date
add_filter( 'slotora_time_slots_for_date', function( $slots, $date ) {
    return $slots;
}, 10, 2 );

// Fires after a booking is saved — hook SMS / WhatsApp here
add_action( 'slotora_after_booking_saved', function( $order_id, $date, $slot ) {
    // ...
}, 10, 3 );

// Detect whether Pro is active
$is_pro = apply_filters( 'slotora_pro_is_active', false );

Additional capability flags — slotora_pro_sms_enabled, slotora_pro_whatsapp_enabled, slotora_pro_delivery_zones_enabled, slotora_pro_multistore_enabled, and slotora_pro_export_enabled — all return false in free and are switched on by Pro.

Questions

Frequently asked

Does this work with block themes and the Gutenberg checkout block?

Yes. Slotora supports both the classic WooCommerce shortcode checkout and the WooCommerce Blocks checkout, registering the date and slot as native Additional Checkout Fields.

Can I make the delivery date optional?

Yes — go to WooCommerce → Delivery Slots → General and turn off the “Required Field” option.

How do I block a public holiday?

Open WooCommerce → Delivery Slots → Blackout Dates and add the date. It's removed from the calendar immediately.

What happens to a slot booking when an order is cancelled?

The booking is released automatically, freeing that slot's capacity for another customer.

Is it compatible with WooCommerce HPOS?

Yes — Slotora declares High-Performance Order Storage compatibility and stores bookings in its own dedicated table.

Why does the slot dropdown say “Select a choose time slot”?

That's WooCommerce Blocks core prepending “Select a ” to your field label. Rename the time-slot label to Time Slot in settings and it reads “Select a time slot.”
